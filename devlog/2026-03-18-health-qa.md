# Health Q&A Skill
**Date**: 2026-03-18  |  **Status**: Completed

## What Was Built
A general health Q&A skill that answers questions about symptoms, conditions, medications, preventive care, mental health, and first aid by fetching credible information from the NIH/NLM MedlinePlus API. An LLM gate function (`needs_health_qa`) classifies whether a user query is a general health question, and if so, the skill searches MedlinePlus, parses the XML response, and injects formatted reference data into the agent's context. Every response includes a medical disclaimer.

## Architecture
User query flows through the reactive agent loop in `agent.py`, which calls `SkillRuntime`. The runtime checks `needs_health_qa()` (GPT-4o binary classifier), then executes `_run_health_qa` which calls `search_health_topics()` (HTTP to MedlinePlus web service) and `format_health_results()` (structured markdown). The formatted output is injected into the LLM context as `HEALTH REFERENCE INFORMATION`, and `instructions.md` governs how the LLM uses it (plain language, disclaimers, no diagnosis, cite sources).

## Technical Decisions
- **MedlinePlus API (no key required)** over paid APIs -- free, authoritative NIH source, XML-based, sufficient for health topic lookup.
- **GPT-4o gate function** with temperature=0 and max_tokens=3 for deterministic YES/NO classification, consistent with other skill gates (workout, nutrition, physical exam).
- **Regex HTML stripping** (`_strip_html`) rather than a full parser -- lightweight, sufficient for MedlinePlus content which uses simple tags. Noted as a known limitation for deeply nested HTML.
- **Summary truncation at 800 chars** with word-boundary split to keep context window usage efficient.
- **No API key / no new dependencies** -- uses only `urllib` and `xml.etree.ElementTree` from stdlib.

## Testing
58 tests across 10 test classes: text helpers (5), XML parsing (7), search with mocked HTTP (5), formatting (8), gate function (3), skill definition (3), executor integration (3), edge cases for all components (19), security/input validation (2). All mocked -- no network calls.

## Known Limitations
- **No validation on `max_results` parameter** -- negative or very large values pass through to the API unchecked.
- **Naive HTML stripping** -- regex-based `_strip_html` may not handle malformed or deeply nested HTML edge cases correctly.
- **No `__init__.py` in tests/** -- test file creates `functions/__init__.py` at import time if missing, but no cleanup of the tests directory itself.
